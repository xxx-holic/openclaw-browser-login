# 小红书平台规则与起号策略 (2025-2026)

## 算法核心

### CES 评分（社区参与度）
```
CES = 点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8
```
评论和转发权重是点赞的 4 倍，关注是 8 倍。**内容要设计成让人想评论和转发的。**

### 流量分发机制
- **搜索流量占 65%+**，是最大流量入口
- 标题前 10 个字权重占 60%+
- 新笔记先进 200-500 人"新手村"测试池
- 30 分钟内 CES 评分决定是否晋级更大流量池
- 5 分钟 / 30 分钟 / 60 分钟赛马机制，与同层级 PK
- 原创性权重 35%
- 推荐流程：召回 → 粗排 → 精排 → 重排（去同质化）

### 搜索算法权重排序
1. 内容相关性（关键词匹配）
2. 内容质量（CES + 完播率）
3. 用户互动数据
4. 账号权重（历史表现）

## 安全红线（触碰即限流/封号）

### 绝对禁止
- 数据造假（买粉、刷赞）
- 直接留微信号/二维码/淘宝口令（OCR 检测图片！）
- 评论区导流（平台零容忍）
- 搬运/抄袭内容
- 虚假人设（豪门/学霸等）
- 电子资源倒卖（PPT模板/盗版书等）
- 医疗内容用"最有效""根治"等绝对化表述

### 高风险行为
- 月删笔记 ≥ 5 篇 → 权重可能归零
- 单设备登录 > 3 个账号 → 限流风险
- 矩阵号同 IP 发同质化内容 → 集体限流
- 频繁修改已发布笔记 → 降权
- 短时间高强度发布 → 识别为刷屏
- 断更 > 1 周 → 不活跃降权
- 频繁切换垂直领域 → 账号权重衰减

### AI 内容规则（2026 新规）
- **所有 AI 生成内容必须标注"AI生成"**
- 纯 AI 生成图片需加专属标签
- AI 辅助创作文字也需标注
- 未标注 → 判定违规
- 真实体验笔记的商业报备通过率提升 60%

## 起号策略

### 第一周：养号期
- 每天模拟真实用户行为 30 分钟+
- 浏览同行笔记、点赞、评论、收藏
- 让平台认定为真实活跃用户
- **不发任何笔记**

### 账号设置
- 头像：真人半身照
- 昵称：带细分领域关键词
- 简介：突出核心价值，不留联系方式
- 垂直细分定位（例：平价彩妆、应届生求职）

### 内容策略
- 每周 3-5 篇高质量笔记（不是越多越好）
- 图文 600 字+ 原创度 ≥ 60%
- "3+2+1"结构：3 个价值点 + 2 个互动引导 + 1 个转化入口
- "活人感" > 精修感（真实生活记录优先）
- 软性种草占比 > 75%，口语化表达

### 标题与封面
- 标题前 18 字包含 2 个核心关键词
- 用冲突感或八卦式提问激发点击
- 封面精美，展示亮点或布局关键词
- 关键词密度 3%-5%，严禁堆砌

### 关键词矩阵
```
搜索框默认词 → 热搜词 → 联想词 → 筛选：
优先选"竞争度小 + 流量大"的长尾词
布局位置：标题 + 正文前100字 + 话题标签 + 评论区
```

### 互动设计（提升 CES）
- 笔记内预埋互动钩子："评论区告诉我你的看法"
- 积极回复评论（前 30 分钟尤其重要）
- "@你的XX朋友" 引导转发
- 视频前 5 秒设强钩子，中段设互动引导，结尾设收藏诱因

### 变现路径
- 1000 粉 → 蒲公英平台接广告
- 品牌合作必须报备（带"品牌合作"标签）
- 引流走私信后台，不在公开区域留联系方式

## AI 自动化安全策略

### 发帖方式选择
| 方式 | 安全性 | 说明 |
|------|--------|------|
| App ADB 操作 | ⭐⭐⭐ 高 | 真机模拟真人点击路径 |
| Web 创作平台 | ⭐⭐ 中 | 非正常用户操作路径 |
| API 直接调用 | ⭐ 低 | 最容易被检测 |

### 发帖节奏
- 每天 2-3 篇，间隔 4 小时+
- 随机化发布时间（不要固定到分钟）
- 静默时段（23:00-7:00）不发帖
- 发布后 30 分钟内人工互动（回复评论）

### 内容安全
- 发布前用违禁词检测工具自查
- AI 写的内容必须人工润色，加入个人口癖
- 配图避免纯 AI 生成（文字会变乱码）
- 用 `--negative-prompt "text, letters, words"` 排除文字

### 限流自查
- 创作者服务中心 → 观众来源分析
- 首页推荐流量占比骤降 = 可能限流
- 搜索账号名看笔记是否被收录
- 检查通知中心有无违规提醒
